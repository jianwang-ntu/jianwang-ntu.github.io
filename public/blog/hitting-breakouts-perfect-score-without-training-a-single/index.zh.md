# 不训练神经网络也能打满分:翁嘉义如何用纯代码刷爆 Atari

![Hitting Breakout's Perfect Score Without Training a Single Neuron](/images/blog/hitting-breakouts-perfect-score-without-training-a-single.png)

视频作者 "为什么叫 QQ" 在这期硬核拆解中介绍了一个反直觉的发现:OpenAI Post Training 团队的研究工程师**翁嘉义**没有训练任何神经网络,仅靠让 Coding Agent 反复看失败回放、改代码、跑测试,就把 Atari 打砖块(Breakout)从平庸分数推到了 864 分的理论上限。视频主张,这不是手写规则的怀旧复辟,而是 Coding Agent 把"启发式规则"的维护成本曲线彻底改写之后,强化学习可能出现的新范式。

## 一个副业实验,意外刷出满分

视频介绍说,翁嘉义清华本科、CMU 硕士,曾在清华 TSAIL 实验室与新加坡 CI Lab 做强化学习,是开源强化学习库 Tianshou 和超快环境引擎 EnvPool 的作者,也参与了 ChatGPT/GPT 系列的 RLHF 基础设施建设。

按视频的说法,这件事最初只是一个工程上的小需求。翁嘉义在业余维护 EnvPool 时想写一些"便宜、可复现"的启发式规则,用来在 CI 里验证游戏环境的正确性,避免每次都得动用昂贵的神经网络。于是他让 Codex 写一个不依赖任何网络的纯代码版本——结果远超预期。

视频列出的成绩单是这样的:

- **Atari Breakout**:策略得分从 387 一路爬到 507、839,最后达到 **864 分的理论最高分**。
- **MuJoCo Ant**:纯代码程序先学会步态、再叠加短视窗模型规划,拿下 **6000 多分**,直接进入主流深度强化学习的量级。
- **MuJoCo HalfCheetah**:用可解释的步态与姿态规则,叠加到 **11,836.7 分**,据视频描述为五倍于常见记录。
- **Vizdoom**(第一人称射击):只用 OpenCV"搓"屏幕视觉,不训练任何网络,平均得分 **557**。
- **Atari 57 全套**:在固定环境交互步数下,纯代码策略的中位数得分在 100 万分附近,**远高于 PPO 等主流深度强化学习算法**。

视频强调,这一切都没有反向传播,没有梯度下降,只有一份在 Coding Agent 手里不断生长的代码。

## 启发式学习:把"更新参数"换成"更新代码"

视频把这套做法归纳为翁嘉义所提出的**启发式学习(Heuristic Learning,HL)**。它的骨架与深度强化学习类似——同样有状态、动作、反馈、更新——但每一个环节都被替换成了软件工程的对象:

- **策略**不再是网络权重,而是**代码**:状态机、控制逻辑,甚至模型预测控制(MPC)。
- **反馈**不再只是一个 reward 数字,而是**测试用例、环境日志、失败视频回放**。
- **更新**不再是反向传播,而是**Coding Agent 直接改代码**。

被长期维护的那个对象,视频按翁嘉义的命名叫作**启发式系统(Heuristic System,HS)**,它不是一份孤立的策略文件,而是包含状态表示、反馈入口、实验记录、测试用例与记忆机制的一整个软件系统。"单条规则不够规则,反馈、历史和下一轮更新全部接起来。"

### 三个被视频反复强调的优势

视频认为,HL 相对深度强化学习有几个尤其诱人的特性:

- **可解释性**:策略是可以一行行读懂的代码,对自动驾驶、医疗决策这类需要可信度的场景意义重大。
- **样本效率**:一次有效的代码更新就是一次质的飞跃,不必靠百万乃至十亿级的环境交互慢慢爬坡。
- **对抗灾难性遗忘**:旧能力不必全靠模型自己"死记硬背",可以**固化成回归测试、固定种子的回放、或写明的失败方向**;新代码一旦破坏旧场景,测试立刻报警。

视频把这一点形容为"给系统装上了一个记忆装置"。

## 为什么是现在:Coding Agent 改写了维护成本曲线

视频对一个反对意见做了正面回应:专家系统、手写规则不是早就被判过死刑了吗?为什么以前没人做?

按视频的解释,过去的瓶颈不是"规则不行",而是**人类维护规则太贵**——今天加一个 `if` 修好一个 bug,明天另一个场景崩了,几周后变成谁都不敢碰的"屎山",每一次修改都可能引发连锁反应。

变量是 Coding Agent。视频用了一个比喻:以前靠人维护规则像工业革命前的手工纺纱,规模一大人就被压垮;现在的 Coding Agent 像不知疲倦的纺织机,可以持续读日志、看回放、改代码、跑测试,变成一条"输送智力的营养管道"。视频强调,**关键的不是规则本身变强了,而是规则的生产与维护成本曲线变了**——过去因维护成本太高而不值得写的规则,现在突然值得长期拥有。

## 把"持续学习"从参数难题变回工程问题

视频借此引出一个更大的论断:持续学习(continual learning)这一长期被锁在"如何更新参数"死胡同里的难题,可以被重述为一个工程问题——**如何维护一个能持续吸收反馈的软件系统**。

不过视频也提醒,一个健康的启发式系统**不能只增长不压缩**,否则即便 Coding Agent 也会陷入"自己都维护不动的屎山"。它必须能定期把一堆局部补丁重构为更简单、更优雅的代码表示——视频将其类比为代码的新陈代谢。

由此带出了一个核心概念:**耦合复杂度**。视频引用翁嘉义的观点:一个系统能走多远,取决于 Coding Agent 能处理多复杂的耦合。良好的模块化、完善的测试覆盖、清晰的日志,都能把认知负担从 Agent 转移到外部工具上。

Breakout 之所以能跑到 864 分,既因为规则本身相对简单,也因为**失败可以视频回放、局部复现、回归验证**;Ant 的策略更复杂,但可以拆成步态、姿态、接触判断、residual MPC 等独立模块,每一块都能独立测试、独立改进。

## 它会取代神经网络吗?

视频明确否认了这种取代叙事。纯代码在复杂视觉感知和长程泛化上依然力不从心——不可能用 `if-else` 写出一个 ImageNet 分类器。

视频认为更值得期待的是**混合架构**,并按翁嘉义的设想给出一个机器人控制的分层例子:

- 用**专用浅层神经网络**负责底层快速感知和状态估计;
- 用 **HL** 负责吸收最新数据、维护安全边界、做局部恢复;
- 用**大语言模型 Agent** 作为"大脑"提供高层反馈,并周期性地把 HL 生成的高质量数据回灌进自己。

视频进一步把这一思路展开成层级:关节级 HL、肢体级 HL、全身平衡 HL、任务级 HL,低层负责安全与低延迟控制,中层负责步态与接触,高层负责任务恢复与长期记忆。在这个图景里,Coding Agent 不必"懂得走路",它更像是插进系统里的**更新管线**——持续把失败视频、传感器流、仿真结果、测试输出喂进系统,再把反馈改写成代码、参数、保护规则与记忆条目。

## 一个被视频反复念出的结论

视频以翁嘉义的那句话作结:**"凡是可以被持续迭代的,都开始能被解决。"** 在作者看来,这不只是一个算法突破,更是一种思维方式的迁移——从更新梯度,到更新代码;从"训出一个聪明的网络",到"维护一个会越来越聪明的软件系统"。视频最后把问题抛回观众:未来的 AI 系统里,纯代码规则与神经网络的比例,会是多少?
