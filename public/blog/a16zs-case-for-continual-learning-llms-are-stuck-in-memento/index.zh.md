# 困在《记忆碎片》里的大模型：a16z 为何押注"持续学习"

![A16Z's Case for Continual Learning: LLMs Are Stuck in *Memento*](/images/blog/a16zs-case-for-continual-learning-llms-are-stuck-in-memento.png)

a16z 的 Malika Aubakirova 与 Matt Bornstein 在 2026 年 4 月发表的长文中，把今天的大语言模型比作 Christopher Nolan 电影《记忆碎片》中患有顺行性遗忘症的 Leonard：能力惊人，却无法形成新的记忆，只能依赖纹身和宝丽来照片维系一个"永恒的当下"。两位作者认为，仅靠上下文窗口、检索系统与系统提示这套"外部脚手架"远远不够——要让 AI 真正进步，必须允许它在部署之后也能像训练时那样，把新经验直接压缩进权重。

## 上下文学习确实有效，但可能正在触顶

文章首先承认：上下文学习（ICL）目前仍在赢。Transformer 本质上是序列上的条件下一词预测器，给它正确的输入序列，就能在不动权重的情况下展现丰富行为。作者引用 Cursor 团队近期关于自主编码 agent 的复盘作为佐证：

> "系统行为中令人意外地大一部分，归结于我们如何提示 agent。Harness 和模型重要，但提示更重要。"

OpenClaw 同样被举为例证——它的突围不靠特殊模型权限（底层模型人人可用），而靠把上下文与工具组织成可工作的状态。两位作者据此提出："看似拙劣却原生"的接口往往胜出，因为它直接耦合到底层系统，而不是与之对抗。

随之而来的是状态空间模型（SSM）与线性注意力变体：把固定记忆层与常规注意力交错堆叠，把 agent 能保持连贯的步数从约 20 步推到约 20,000 步。作者称这"在某种意义上也是一种持续学习"——虽然没更新权重，但引入了一个几乎不需要重置的外部记忆层。

## "归档柜谬误"

文章的核心论点借用了 Ilya Sutskever 反复强调的看法——LLM 本质上是压缩算法。训练阶段把互联网压缩进参数，正是这种**有损压缩**逼迫模型寻找结构、抽象出可迁移的表征。作者写道：让 LLM 强大的机制（把原始数据压缩成紧凑、可迁移的表征），恰恰是部署后我们拒绝让它继续做的事。

他们用一个思想实验点出"归档柜谬误"：一个拥有无限存储、能即时检索任何事实的系统，算"学会"了吗？没有——因为它从未被迫做压缩。

为说明 ICL 的局限，作者引用 Yu Sun 给出的费马大定理例子：350 年间无人证出，并非因为缺少文献，而是因为答案在概念上与既有数学之间的距离过远；Andrew Wiles 在近乎与世隔绝的七年里发明了新技术，最终才把椭圆曲线与模形式两支数学桥接起来。LLM 是缺了一种更新先验、做真正创造性思考的能力，还是说人类知识不过是更大尺度上可被训练 / 重组的数据？文章坦承：这是经验问题，目前没有答案。

作者还指出 ICL 的另一道天花板：语言难以表达的高维、隐性、深层结构——医学影像中区分良性伪影与肿瘤的视觉纹理、定义说话人独特韵律的音频微波动——再长的上下文窗口也装不下，只能存在于参数里。这也帮助解释了为什么 ChatGPT 的"记忆"功能常常让用户不安：用户要的不是"被记住"，而是能力——能内化你模式的模型可以泛化到新情境，仅仅回放历史的模型不能。

## 持续学习的光谱：上下文 / 模块 / 权重

两位作者把研究路线沿一条光谱展开，分界不在于"有没有记忆功能"，而在于**压缩发生在哪里**：

- **上下文端**：更聪明的检索管线、agent harness、提示编排。最成熟，瓶颈在深度。多 agent 架构被列为该端的延伸——一群各自持有上下文、各自专精切片的 agent，可以近似无界工作记忆，Karpathy 近期的 autoresearch 项目与 Cursor 构建浏览器的案例被点名为早期范式。
- **模块端**：可拆挂的知识模块（压缩 KV cache、适配层、外部记忆），让 8B 模型在目标任务上以一小部分显存匹敌 109B 模型。
- **权重端**：稀疏记忆层只更新相关参数；强化学习闭环；以及把上下文在推理时压缩进权重的 test-time training（Sun 等 2020 起，至今演化出 TTT layers、TTT-E2E、TTT-Discover）。文章还罗列了一条平行脉络：EWC（Kirkpatrick et al., 2017）的正则化、MAML（Finn et al., 2017）与 Nested Learning（Behrouz et al., 2025）的元学习、LoRD 与自蒸馏 SDFT 的高效蒸馏，以及 STaR、AlphaEvolve、Silver 与 Sutton 的"经验时代"所代表的递归自我改进。

## 为什么"天真地更新权重"会失败

文章特别用一节列出尚未在规模上解决的失败模式：灾难性遗忘（稳定—可塑性两难）、时间错位（不变规则与可变状态被压进同一组权重）、逻辑整合失败（事实更新不传播到其推论）、以及"减法不可微"导致的不可遗忘问题。

更少被讨论的是第二组问题：训练与部署的分离本身就是安全、可审计与治理的边界。一旦打开，安全对齐可能不可预测地退化；持续更新会形成一种活在权重里的、缓慢而持久的数据投毒面；持续变动的模型无法版本化、无法回归测试、也难以一次性认证；用户交互被压缩进参数还会放大隐私风险。作者强调，这些是开放问题而非根本不可能，解决它们与攻克架构难题同样属于持续学习的研究议程。

## 收束

回到 Memento 的隐喻：作者写道，Leonard 的悲剧不在于无法运转——他在每一幕里都足智多谋——而在于**他永远无法累积**。今天的 AI 也受困于同样的约束：检索系统再强大，也不是学习；查得到任何事实的系统，并未被迫去发现结构、被迫去泛化。

文章给出的判断是：前路大概不是某个单点突破，而是一个分层系统。ICL 仍将是适应的第一线，模块机制处理个性化与领域专化的中间地带；但对于真正困难的问题——发现、对抗性适应、无法用语言表达的隐性知识——可能需要模型在训练之后，仍能把经验压缩进自己的参数。否则，正如两位作者收尾时所说，归档柜会越来越大，但归档柜终究只是归档柜——我们只会困在自己的《记忆碎片》里。
