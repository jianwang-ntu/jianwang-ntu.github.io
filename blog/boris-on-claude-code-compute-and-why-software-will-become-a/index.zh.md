# Claude Code 创造者 Boris：编程将成为像识字一样的基础技能

![Boris on Claude Code, Compute, and Why Software Will Become a Basic Literacy](https://publicsg.s3.ap-southeast-1.amazonaws.com/github.io/images/blog/boris-on-claude-code-compute-and-why-software-will-become-a.png)

在 Anthropic 开发者大会的炉边谈话中，Claude Code 的创造者 Boris 接受了主持人 Kate 的采访。他提出一个核心论断：当下正经历一场堪比"印刷术问世"的变革——AI 代理（agent）已经在重塑软件开发，而真正能享受到生产力红利的企业，必须像 1990 年代那样重新围绕这项新技术重构整个业务流程。

## 从 Instagram 到 Anthropic Labs

Boris 简单回顾了自己的从业轨迹：早年长期混迹于初创公司，曾作为第一位工程师加入 Y Combinator 第二或第三批的某家创业公司；之后在 Meta 工作了七年，担任 Instagram 的技术负责人之一，并负责整个公司代码库的代码质量。

他说自己当时人在日本，意识到"一件彻底疯狂的事情正在发生"，认为这项游戏规则改变级别的技术必须被安全地、以正确方式做出来，于是加入了 Anthropic。招他进来的是联合创始人 Ben，团队归 Mike Krieger（Instagram 联合创始人之一）领导，正是那支后来被解散又重新组建的 Anthropic Labs。Boris 介绍说，这支小团队前后产出了四件东西：**Claude Code、MCP、Skills 和 Desktop App**。在他看来，Anthropic 多年来对企业、安全和编程的聚焦，自然导向了这些产品。

## 大会发布：没有新模型，但有更多算力

谈到当天的发布，Boris 直言今天没有模型公告，也没有重磅产品发布。重点放在两件事上：

- **托管代理（Managed Agent）API**：让开发者更轻松地在 Anthropic 平台上构建 agent，并提供配套工具。
- **算力扩张**：宣布与 SpaceX 合作，**Colossus 1** 集群将专门用于服务 Anthropic 的客户。

他将这件事放在公司整体增长的语境下解释：增长一直在超出公司自己的所有预测，而 Anthropic 一贯把"对算力负责任地管理"当作核心原则——既不超买，也不少买。算力分配是产品和研究之间的长期平衡问题，而不是"找 Dario 抢资源"的短期博弈。

## "Co-work"：写代码之外的 Claude Code

Boris 把"agent 化"形容为新一代工作方式：他自己在任何时刻都有"几个、有时上千个 agent"在替他干活，他可以专心于想点子、和客户交流，把不喜欢做的事交出去。

他讲了 Co-work 这个新产品的来历。Claude Code 发布大约半年后，团队发现用户开始把它用于**非编程任务**：数据分析、项目管理，甚至有人在 Twitter 上展示用它接监控摄像头来监测番茄植株的养分、辅助种植。Boris 说，作为产品人，看到用户自发把产品用在新场景上是"最酷的事情"，这就是市场需求在告诉团队该做什么。Co-work 正是回应这种需求的产品，他声称其增长比 Claude Code 早期还要快。

## 为什么大多数公司还没吃到红利

Kate 追问"为什么美国大企业普遍还没看到这种生产力提升"，Boris 借用一份哈佛商学院 1990 年代初的案例研究作答：那篇文章讨论的是"电脑都已经摆在办公室里了，为什么生产力没起来"，结论是企业必须把电脑放在业务流程的中心，而不是把它塞在角落、继续用旧的文件柜办公。

他声称在 Anthropic 内部以及最成熟的客户那里，正在发生同样的事——把 Claude 放到所有事务的中心。当业务被这样重构后，看到的生产力提升不是过去他做开发者效率工作时熟悉的"几个百分点"，而是**几百个百分点**，而且增长速度还在加快。他举了 Shopify 等企业客户，以及 NASA 用 Claude Code 帮助绘制火星车路径规划作为例子，并预计知识工作领域出现这种转变还需要 6 到 12 个月。

## 对就业、护城河和应用商店的看法

面对就业冲击的疑问，Boris 把回答限定在短期：眼下他看到的是"机会"，人们被从厌倦的劳动中解放出来。他自己作为工程师摆脱的是调试和"硬想程序怎么跑"的负担；他还顺手举例说，自己出差的机票就是 Co-work 帮他订完的。

谈到软件行业的护城河，他认为有些壁垒确实在消失——比如**切换成本**，因为 Claude 可以直接帮用户写软件，从一个产品迁到另一个产品变得容易。但他也表示，大部分老牌护城河依然成立，只是一切节奏都被加速了。

至于"Anthropic 是否要做用户的总入口"，Boris 的回答是分两条腿走路：自己想拥有部分客户关系，但同时坚持公平——团队自己用的整套技术栈（Claude Agent SDK、Anthropic API）都开放给开发者，已经有数以千计的开发者在和 Anthropic 内部用同一套基础设施上构建公司。被问到这是否意味着应用商店会被颠覆、应用会变成实时生成时，他坦承"这是个好问题"，希望和做应用商店的人聊聊。

## 编程将像识字一样普及

Boris 自己依然写代码，但形态已经完全变了。他描述说，大约半年前起，自己原本手写的代码现在全部由 Claude 生成；他与 Claude 对话——"嘿，咱们做这个功能"——Claude 编写、测试、展示，他给出反馈或确认。他把这一转变追溯到去年 11 月发布的 Opus 4.5，并称其在 Opus 4.6 和 4.7 时进一步加速，对越来越多的开发者而言，这就是今天编程的样子。

那么软件开发者这个职业还存在吗？他给出了一个鲜明的类比：

> 在 1400 年代，大多数人不识字；今天大多数人识字。我预测**编写软件会变成像读写一样的基础技能**。每个人都会通过和 Claude 对话写出东西，但社会仍然需要专业写作者，正如它需要专业的程序员。

Kate 把这总结为"印刷术时刻"，Boris 表示认同。

## 安全、Mythos 与人才之争

关于安全，他承认最近三个月内出现了一次"相位变化"：模型在各类软件中**寻找漏洞的能力变得异常强大**，威胁模型也随之改写。Anthropic 的应对思路是确保"好人先拿到最好的模型"——好让防御者能赶在攻击者之前发现并修补漏洞。被问到 Mythos（Anthropic 内部的安全研究项目）是否会更广泛地开放，他直接回答：**目前没有这样的计划**。

对于"AI 是否会降低对顶级工程师的需求"，Boris 给出的答案是否定的——无论是工程师、产品经理还是设计师，找最优秀的人依旧关键。他也重申，开发者、编程和企业一直是 Anthropic 的核心焦点，并不会改变。

## 给学生的建议：拥抱工具，去开公司

谈话最后，Kate 问他会给一个今天大三的经济学专业学生什么建议（Boris 自己本科念的是经济学）。他给出两条：

1. **学会用这些工具**。不要害怕 agent，不要害怕 AI，主动尝试，看哪些方式适合自己。
2. **如果有创业的冲动，就去创业**。他声称这是"史上最适合做创业公司的时刻"，并预测十年内创业公司的数量会比今天多 10 倍乃至 100 倍。

谈话至此，Boris 描绘的图景是一个被 agent 重构的工作世界——前提是企业、个人和社会都愿意像几百年前学会读写、几十年前学会用电脑那样，重新组织自己的工作方式。
